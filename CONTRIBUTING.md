# Contributing

Thank you for considering contributing to Bootstrap System.css Theme!

## Guidelines

* This project follows standard HTML/CSS/JavaScript best practices
* Any contribution should maintain compatibility with Bootstrap 5.3.8
* Test your changes across major browsers (Chrome, Firefox, Safari)
* Keep the retro aesthetic faithful to Apple System OS (1984-1991)
* Pull requests must be sent from a new feature branch, not from `main` or `master`

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
* A clear description of the problem
* Steps to reproduce
* Expected vs. actual behavior
* Screenshots if applicable
* Browser and version information

### Suggesting Enhancements

Open an issue describing:
* The enhancement you'd like to see
* Why it would be useful
* How it should work (if you have ideas)

### Submitting Pull Requests

1. Fork the repository
2. Create a new branch for your feature/fix:
   ```sh
   git checkout -b feature/my-new-feature
   ```
3. Make your changes
4. Test thoroughly in multiple browsers
5. Commit with clear, descriptive messages
6. Push to your fork
7. Open a Pull Request with a clear description

## Development Setup

1. Clone the repository:
   ```sh
   git clone https://github.com/YOUR_USERNAME/bootstrap-system-css-theme.git
   cd bootstrap-system-css-theme
   ```

2. Open `index.html` in your browser to see the demo

3. Edit `css/bootstrap-system-theme.css` to make theme changes

4. Refresh your browser to see changes

## Code Style

* Use consistent indentation (4 spaces)
* Add comments for complex CSS rules
* Keep selectors specific but not overly complex
* Maintain alphabetical ordering of CSS properties where practical

## Testing Checklist

Before submitting a PR, verify:
- [ ] Changes work in Chrome/Edge
- [ ] Changes work in Firefox
- [ ] Changes work in Safari
- [ ] No console errors
- [ ] `index.html` demo displays correctly
- [ ] Retro aesthetic is preserved
- [ ] Bootstrap functionality still works
